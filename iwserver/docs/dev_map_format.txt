Map File Format (Developer Reference)

Map files are plain text stored in iwserver/map/ with the .map extension.

Header fields (one per line):
mapname:[name] - The display name of the map.
maxx:[value] - Maximum X coordinate.
maxy:[value] - Maximum Y coordinate.
maxz:[value] - Maximum Z coordinate.

Tile definitions:
tile:x1:x2:y1:y2:z1:z2:tilename - Defines a walkable tile in a rectangular region.

Zone definitions:
zone:x1:x2:y1:y2:z1:z2:zonename - Defines a named zone for player orientation. Zone names are shown when pressing B.

Ambience definitions:
ambience:x1:x2:y1:y2:z1:z2:soundfile - Plays a looping sound in the specified region.

Ladder definitions:
ladder:x:y:z1:z2:mapname - A climbable ladder at the given position connecting two Z levels.

Point of interest definitions:
poi:x:y:z:name - A named location listed by the points-of-interest menu (J).

Turbolift definitions:
turbolift:x1:x2:y1:y2:z1:z2:liftname:level1,z1:level2,z2:... - Declares a turbolift whose control panel occupies the given region (spanning z1..z2 across every level it serves). The trailing fields each name a destination level and its absolute Z, separated by a comma. Level names must not contain a colon or a comma. When a player stands in the panel region and presses Shift+P, a menu offers those levels; selecting one moves the player to that level's Z, announcing every served level passed along the way. Parsed client-side in includes/map.nvgt.

The map system is initialized with init_mapsystem() on the server. Maps are sent to clients as raw text via the mapdata message.
