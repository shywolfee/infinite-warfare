New in 0.3
Fixed the administrator panel so developers running from source can actually open it.
  Previously, the panel verified access using the computer ID against admins.txt, which
  meant developers who weren't listed as admins were locked out of the panel they could see.
The changelog viewer is now a form with a read-only text field and a close button.
Account-related options (login, authorization code, password reset) have been moved out of
  the options menu into a dedicated Account menu, which requires the server to be online.
  The options menu is now cleaner, containing only game settings like duo mate and right control.
Added a password reset feature.
  Players can request a password reset from the account menu by entering their username.
  A reset code is generated and stored on their account.
  Administrators can view pending password reset requests and codes in the admin panel.
  When the player enters their reset code, they are prompted to set a new password.

New in 0.2.1
Fixed a compilation error caused by calling string_trim_whitespace, a function that does not
  exist in NVGT. The code confidently called it in two places as if it were a real function,
  checking if server responses were blank. The empty string check alone was already sufficient
  since the server sends controlled data, making the entire call both broken and unnecessary.

New in 0.21
Added an Administrator Panel to the main menu (only visible when running from source).
  The admin panel connects to the server to verify administrator access.
  Authorization mode: a checkbox that, when enabled, requires new accounts to be approved by an admin before they can log in.
  View unauthenticated players: shows a list of players awaiting approval along with their authorization codes.
When authorization mode is enabled, newly registered accounts are created but cannot log in.
  A message tells the player that their account must be authenticated first.
  Administrators can view the pending player and their authorization code in the admin panel, then provide the code to the player.
  A new "Authorization code" field and "Authorize account" button have been added to the options menu.
  Players enter their username and the code provided by the admin to authenticate their account.
Players now respawn in the lobby instead of reconnecting to the server when they die.
Exiting the game now returns you to the lobby instead of the main menu, keeping your connection alive.

New in 0.2
The options menu is now a proper form with input fields, a checkbox, and buttons instead of a list menu.
  Account login: enter your username and password directly in the form and press Login.
  Duo mate: type your duo mate name and press Set duo mate.
  Right control detection: toggle the checkbox and press Save to apply.
The lobby and all its tabs now use forms instead of list menus.
  Account Info tab shows your details in read-only fields.
  Players tab displays the player list in a form.
  Rewards tab presents each milestone in its own field.
  Modes tab uses buttons to select a game mode.
Fixed a bug where map ambience would play when entering the lobby before joining a game.
Players now receive a message when someone enters or leaves the lobby.
Players now receive a message when someone enters a game mode from the lobby.
Press F1 in the lobby to open the player list showing every connected player's name, dev or admin status, and whether they are in the lobby or in a game mode.

New in 0.19
Connecting to the server now takes you to a lobby instead of dropping you straight into the game.
The lobby has four tabs navigated with the arrow keys:
  Modes: lists available game modes. Currently Battlegrounds is available.
    Select Battlegrounds and press Enter to deploy to the main map and begin playing.
  Account Info: displays your username, email, registration status, and duo mate.
  Players: queries the server for the live in-game player list and reads it aloud.
  Rewards: describes the kill-reward milestones (3 kills and 5 kills).
Leaving the lobby returns you to the main menu without entering the game world.
The server now handles the lobby_players request from pre-spawn clients.

New in 0.18
Weapons now use a realistic magazine and clip-based ammo system.
Each weapon has a specific feed device with a real-world capacity and must be reloaded manually.
Press R to reload. The current feed device is ejected and a fresh one is loaded from reserve.
Press Shift+R to unload the current magazine back into your reserve pool.
Press A to hear rounds loaded, ammo type, feed device name, and rounds in reserve.
When a weapon runs dry, you hear the empty click and a reminder to press R.
Firing is blocked while a reload is in progress.
Beta: all players start with a full weapon and generous reserve magazines.
Ammo types and feed devices by weapon:
  Glock 17: 9x19mm Parabellum, 17-round magazine (10 spare)
  Beretta 92FS: 9x19mm Parabellum, 15-round magazine (10 spare)
  Taurus Model 66: .357 Magnum, 6-round cylinder (12 spare)
  Magnum Revolver: .44 Magnum, 6-round cylinder (12 spare)
  FN P90: FN 5.7x28mm, 50-round magazine (6 spare)
  HK G36: 5.56x45mm NATO, 30-round magazine (8 spare)
  AK-47: 7.62x39mm, 30-round magazine (8 spare)
  TAR-21: 5.56x45mm NATO, 30-round magazine (8 spare)
  FAMAS F1: 5.56x45mm NATO, 25-round magazine (8 spare)
  Benelli M4: 12-gauge buckshot, 8-round tube magazine (8 spare)
  KS-123: 12-gauge buckshot, 5-round tube magazine (8 spare)
  M1 Garand: .30-06 Springfield, 8-round en-bloc clip (8 spare)
  SG-08 Sniper: .308 Winchester, 10-round magazine (5 spare)
  120mm Mortar: 120mm HE mortar shell, single-load barrel (5 spare)

New in 0.16
The server must now be reachable before you can register a new account or log in via the options menu.
If the server is offline, both actions are blocked with a clear error message.
Added server_online() network probe that checks connectivity before sensitive account operations.

New in 0.15
Added a full account system.
Accounts are stored in iwserver/accounts/ alongside the server data.
Register a new account from the main menu: choose a username, password, confirm the password, and provide a valid email address.
You must read and agree to the server rules before your account is created.
The options menu now has a "Set account (login)" option which replaces the old "Set your name" entry.
Enter your registered username and password to log in and set your in-game name.
Passwords are stored encrypted with a per-account salt.
