New in 0.4
Added 15 new weapons: Desert Eagle, MP5, UZI, M4A1 Carbine, SCAR-H, RPK LMG, M249 SAW, AWP Sniper, Dragunov SVD, SPAS-12, MP7, FN FAL, Crossbow, Katana, Baseball Bat.
Added fire mode system. Press Shift+A to cycle between semi-automatic, burst, and full automatic on supported weapons.
Added stamina system. Stamina degrades when using melee weapons, jumping excessively, or moving with heavy weapons. Restores every 5 seconds naturally.
Added energetic potions in 4 tiers: red, blue, green, and rainbow. Each restores increasing amounts of stamina.
Added buffs and debuffs system. Health potions give Residual Healing (HP restore every 15s for 2 min). Ultra potions give stronger healing (every 25s for 5 min). Smoke bombs apply Blinded debuff.
Added equipment system with body slot. Bomb vests are now equipped as body equipment and can be removed in the equipment menu.
Replaced grave accent game menu with escape menu. New menu includes: changes, rules, account info, developers, game statistics, equipment, account statistics, mode statistics, and exit game.
Press H to see health, stamina, and active buffs/debuffs.
Updated starting loadout pools with all new weapons.

New in 0.35
Added developer suite accessible with Shift+D (dev builds only), featuring a give all items option.
Added documentation system with help entries in iwserver/docs. Press Shift+H to browse all help topics, or Alt+Enter on an inventory item to view its help entry.
Enter key now draws weapons from inventory in addition to using items, matching quickbar behavior.

Added inventory category cycling with J and K keys. Categories: All, Weapons, Ammo, Consumables, Equipment. Tab and Shift+Tab only show items in the selected category.

New in 0.3.4
Completely redesigned the weapon system.
  Weapons are now inventory items.  You can see them when cycling your
  inventory alongside ammo, potions, and other gear.
  Number keys 1-0 are now a quickbar with 10 customizable slots.
  Press Alt and a number key to bind the currently selected inventory
  item to that quickbar slot.  Press the number key by itself to use
  the bound item: weapons are drawn, consumables are used.
  Starting weapons are auto-assigned to quickbar slots 1-5.
Players no longer spawn with all weapons.
  Each spawn gives a random loadout from a pool:
    1 melee weapon (machete or combat knife),
    1 sidearm (Glock 17, Beretta 92FS, or a revolver),
    1 primary weapon (assault rifle, SMG, or shotgun).
  Ammunition is given only for the weapons you actually received.
  You also start with 2 health potions and 3 frag grenades.
  Additional weapons can be found from corpses, chests, and drops.
Added a developer Give All command (grave accent key).
  Only available when running from source (not in compiled builds).
  Gives every weapon, all ammunition types, and all usable items.

New in 0.3.3
Fixed Q, E, S and X keys not working without Shift.
  The key press was consumed by the Shift check and never reached the
  normal handler.  All four keys now cache the press in a variable first.
Refactored the ammunition system to use inventory items for reserves.
  Magazine-fed weapons (pistols, rifles, SMGs) consume a magazine item
  from your inventory when you reload.  Unloading returns the magazine.
  Revolvers draw individual rounds from an ammo belt carried in your
  inventory.  Reloading pulls rounds from the belt into the cylinder.
  Shotguns load individual shells from a shell box in your inventory.
  The M1 Garand uses en-bloc clips and the mortar uses loose shells.
  Shared ammo types work across weapons (e.g. STANAG magazines are
  used by both the HK G36 and TAR-21).
All reserve ammunition is now visible when cycling your inventory.
Beta: players start with all weapons and generous reserve ammo.

New in 0.3.2
Added fine-grained turning controls with the Shift key.
  Shift+Q: turn left by 1 degree (instead of the normal 45-degree snap).
  Shift+E: turn right by 1 degree.
  Shift+S: aim up by 1 degree.
  Shift+X: aim down by 1 degree.
  Without Shift, Q/E/S/X still snap to the major positions as before.
Vertical aiming now uses a degree-based system internally (0 to 180 degrees).
  0 degrees is straight down, 90 degrees is straight forward, 180 degrees is straight up.
  The server weapon system has been updated to use range-based angle checks to match.
Added new options to the options menu.
  Camera speed: controls how fast the camera moves when holding arrow keys (1 to 200).
  Master volume: adjusts the overall game volume (-60 to 0 dB).
  Both settings are saved to your preferences and persist between sessions.

New in 0.3.1
Fixed a bug where unauthenticated players could bypass the authorization check and connect
  to the server. The connect function only checked whether a name was set, but the name could
  be loaded from saved preferences without going through the account login flow. An explicit
  authentication check is now performed before connecting.
The account menu has been reorganized into separate sub-menus for clarity.
  Instead of one cluttered form with login fields, authorization code, and password reset all
  mixed together, there is now a hub menu with three options:
  Login: enter your username and password to log in.
  Authorize account: enter your username and the authorization code from an administrator.
  Password reset: request a reset, then enter the reset code to set a new password.

New in 0.3
Fixed the administrator panel so developers running from source can actually open it.
  Previously, the panel verified access using the computer ID against admins.txt, which
  meant developers who weren't listed as admins were locked out of the panel they could see.
The changelog viewer is now a form with a read-only text field and a close button.
Account-related options (login, authorization code, password reset) have been moved out of
  the options menu into a dedicated Account menu, which requires the server to be online.
  The options menu is now cleaner, containing only game settings like duo mate and right control.
Added a password reset feature.
  Players can request a password reset from the account menu by entering their username.
  A reset code is generated and stored on their account.
  Administrators can view pending password reset requests and codes in the admin panel.
  When the player enters their reset code, they are prompted to set a new password.

New in 0.2.1
Fixed a compilation error caused by calling string_trim_whitespace, a function that does not
  exist in NVGT. The code confidently called it in two places as if it were a real function,
  checking if server responses were blank. The empty string check alone was already sufficient
  since the server sends controlled data, making the entire call both broken and unnecessary.

New in 0.21
Added an Administrator Panel to the main menu (only visible when running from source).
  The admin panel connects to the server to verify administrator access.
  Authorization mode: a checkbox that, when enabled, requires new accounts to be approved by an admin before they can log in.

Players now respawn in the lobby instead of reconnecting to the server when they die.

New in 0.18
Weapons now use a realistic magazine and clip-based ammo system.
Each weapon has a specific feed device with a real-world capacity and must be reloaded manually.
Press R to reload. The current feed device is ejected and a fresh one is loaded from reserve.
Press Shift+R to unload the current magazine back into your reserve pool.
Press A to hear rounds loaded, ammo type, feed device name, and rounds in reserve.
When a weapon runs dry, you hear the empty click and a reminder to press R.
Firing is blocked while a reload is in progress.
Beta: all players start with a full weapon and generous reserve magazines.
