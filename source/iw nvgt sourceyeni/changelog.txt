New in 0.452, build 6 (2026-03-17)
Added Habitat Alpha, a new game mode. Similar to Battlegrounds but set on a sci-fi map: an abandoned colony on a distant world far from Earth.
The map features several city/district and administrative domes connected by underground tunnels, each with a stairwell and staircase leading up to the street (or in admin domes, an access corridor leading into the tunnel network).
Some domes have airlocks that let you travel outside and walk between domes on the surface.
Zones on the outside are highly detailed, telling you which dome you're near, whether you're along the north/south/east/west wall of a dome, near or at the entrance to an airlock, plus paths and roads between domes.

New in 0.45, build 4 (2026-03-17)
Completely rebuilt the main map. The new map is an archipelago of five islands in an ocean connected by bridges.
The central island contains a full city with a central plaza and fountain, city hall, market, armory, barracks, watchtower, hospital, warehouse, and a park.
The north island is a fishing village with three cottages and a dock.
The south island is a town with a town hall, general store, and tavern.
The east island is a residential area with six houses along named streets.
The west island contains a mine with an equipment shed, underground mine tunnels, and ore deposits.
Four bridges (north, south, east, west) connect the outer islands to the central island, with fence railings and ambient water sounds.
Added 5 underground bunkers (Alpha through Echo), one beneath each island. Each bunker has a ladder entrance, control panel, and item spawns.
All five bunkers are connected by an underground tunnel network running north-south and east-west beneath the ocean floor.
The mine on the west island has its own underground tunnel system connecting to bunker Echo.
Map expanded from 500x500 to 1000x1000 to accommodate the island layout.
Renamed map from "main" to "Archipelago".

New in 0.44, build 3 (2026-03-17)
Feature quality-of-life pass.
Overhauled the escape menu. Quick stats (weapon, location, stamina, effects, inventory count, map, player info) are now displayed as read-only fields at the top of the menu instead of being buried in separate statistics sub-menus. Game statistics, account statistics, and mode statistics sub-menus have been removed; all useful info is visible immediately when you open the menu.
Magazines now use real-world caliber names instead of weapon-specific names. For example, "glock_17_magazine" is now "9x19mm 17-round magazine", "stanag_magazine" is now "5.56mm STANAG magazine", and so on for all weapon types.
Weapons that share caliber and magazine capacity now share the same magazine item. SCAR-H and FN FAL both use "7.62x51mm 20-round magazine", and Glock 17 and Beretta 92FS share "9x19mm" magazines with their respective capacities.
Fixed updater: added missing return after a failed version check, which previously caused the update logic to run with invalid data.
Fixed sound file checks in dynamic_menu_pro: corrected inverted logic that attempted to play sounds even when the file did not exist.
Added explicit parentheses to fire control conditions for readability.

Ammo containers (ammo belts, shell boxes, bolt quivers) now display as a single item with round count in parentheses, e.g. ".357 magnum ammo belt (72)" instead of "72 .357 magnum ammo belts".

New in 0.34
Completely redesigned the weapon system.
  Weapons are now inventory items.  You can see them when cycling your
  inventory alongside ammo, potions, and other gear.
  Number keys 1-0 are now a quickbar with 10 customizable slots.
  Press Alt and a number key to bind the currently selected inventory
  item to that quickbar slot.  Press the number key by itself to use
  the bound item: weapons are drawn, consumables are used.
  Starting weapons are auto-assigned to quickbar slots 1-5.
Players no longer spawn with all weapons.
  Each spawn gives a random loadout from a pool:
    1 melee weapon (machete or combat knife),
    1 sidearm (Glock 17, Beretta 92FS, or a revolver),
    1 primary weapon (assault rifle, SMG, or shotgun).
  Ammunition is given only for the weapons you actually received.
  You also start with 2 health potions and 3 frag grenades.
  Additional weapons can be found from corpses, chests, and drops.
Added a developer Give All command (grave accent key).
  Only available when running from source (not in compiled builds).
  Gives every weapon, all ammunition types, and all usable items.

New in 0.33
Fixed Q, E, S and X keys not working without Shift.
  The key press was consumed by the Shift check and never reached the
  normal handler.  All four keys now cache the press in a variable first.
Refactored the ammunition system to use inventory items for reserves.
  Magazine-fed weapons (pistols, rifles, SMGs) consume a magazine item
  from your inventory when you reload.  Unloading returns the magazine.
  Revolvers draw individual rounds from an ammo belt carried in your
  inventory.  Reloading pulls rounds from the belt into the cylinder.
  Shotguns load individual shells from a shell box in your inventory.
  The M1 Garand uses en-bloc clips and the mortar uses loose shells.
  Shared ammo types work across weapons (e.g. STANAG magazines are
  used by both the HK G36 and TAR-21).
All reserve ammunition is now visible when cycling your inventory.
Beta: players start with all weapons and generous reserve ammo.

New in 0.32
Added fine-grained turning controls with the Shift key.
  Shift+Q: turn left by 1 degree (instead of the normal 45-degree snap).
  Shift+E: turn right by 1 degree.
  Shift+S: aim up by 1 degree.
  Shift+X: aim down by 1 degree.
  Without Shift, Q/E/S/X still snap to the major positions as before.
Vertical aiming now uses a degree-based system internally (0 to 180 degrees).
  0 degrees is straight down, 90 degrees is straight forward, 180 degrees is straight up.
  The server weapon system has been updated to use range-based angle checks to match.
Added new options to the options menu.
  Camera speed: controls how fast the camera moves when holding arrow keys (1 to 200).
  Master volume: adjusts the overall game volume (-60 to 0 dB).
  Both settings are saved to your preferences and persist between sessions.

New in 0.31
Fixed a bug where unauthenticated players could bypass the authorization check and connect
  to the server. The connect function only checked whether a name was set, but the name could
  be loaded from saved preferences without going through the account login flow. An explicit
  authentication check is now performed before connecting.
The account menu has been reorganized into separate sub-menus for clarity.

  mixed together, there is now a hub menu with three options:
  Login: enter your username and password to log in.
  Authorize account: enter your username and the authorization code from an administrator.
  Password reset: request a reset, then enter the reset code to set a new password.
